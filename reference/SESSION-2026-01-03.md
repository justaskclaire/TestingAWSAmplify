## ‚ö†Ô∏è Important Notes

### Font Variables in CSS
The layout now provides CSS variables for fonts:
- `--font-inter` - Body text (already defined in source)
- `--font-playfair` - Display headings (already defined in source)

Use them in CSS:
```css
.heading {
  font-family: var(--font-family-display); /* Uses Playfair */
}
```

### Design System Usage
All new components should use CSS custom properties:
```css
.card {
  padding: var(--space-4);
  border-radius: var(--radius-lg);
  box-shadow: var(--shadow-md);
  transition: all var(--transition-base);
}
```

### Component Structure
Following Next.js 14 App Router best practices:
- Server components by default
- Client components only when needed (interactivity)
- CSS modules or separate .css files
- TypeScript for type safety

## üé® Design System Quick Reference

### Colors
- **Primary:** `--color-primary-500` (#f43f5e - rose gold)
- **Accent:** `--color-accent-500` (#a855f7 - lavender)
- **Text:** `--color-text` (gray-900)
- **Surface:** `--color-surface` (white)
- **Border:** `--color-border-light` (subtle)

### Spacing
- `--space-1` to `--space-6` (8px to 48px)
- Use semantic names, not pixel values

### Typography
- **Display:** `--font-family-display` (Playfair Display)
- **Body:** `--font-family` (Inter)
- **Sizes:** `--font-size-xs` to `--font-size-3xl`
